# 微服务架构学习笔记

## 项目概述

这是一个基于 Node.js + TypeScript 的微服务学习项目，使用 pnpm workspace 进行 monorepo 管理。

## 项目架构

### 整体结构

```
.
├── common/              # 公共模块（共享配置和工具）
├── packages/            # 微服务包目录
│   ├── gateway/        # API 网关服务
│   ├── post/           # 帖子服务
│   └── user/           # 用户服务
├── pnpm-workspace.yaml # pnpm 工作区配置
└── tsconfig.json       # TypeScript 配置
```

### 服务端口分配

根据 `common/index.ts` 的配置：

- **User Service**: 3000 端口
- **Gateway Service**: 3001 端口
- **Post Service**: 3002 端口

## 核心概念

### 1. 微服务架构模式

这个项目展示了典型的微服务架构：

- **服务拆分**: 按业务领域划分（用户、帖子）
- **API 网关**: 统一入口，路由转发
- **独立部署**: 每个服务可独立运行和部署

### 2. Monorepo 管理

使用 **pnpm workspace** 管理多个包：

```yaml
# pnpm-workspace.yaml
packages:
    - "common/*"
    - "packages/*"
```

**优势**:
- 代码共享方便（common 模块）
- 统一依赖管理
- 便于本地开发和调试

### 3. 技术栈选择

#### Gateway Service (API 网关)
- **框架**: Fastify
- **特点**: 高性能、低开销
- **职责**: 
  - 请求路由和转发
  - 统一鉴权
  - 负载均衡
  - API 聚合

#### Post Service (帖子服务)
- **框架**: Express
- **特点**: 成熟稳定、生态丰富
- **职责**:
  - 帖子的增删改查
  - 帖子相关业务逻辑

#### User Service (用户服务)
- **框架**: 待实现
- **职责**:
  - 用户注册登录
  - 用户信息管理
  - 身份认证

### 4. Common 模块

**作用**: 存放各服务共享的代码

当前包含：
- `servicePort`: 服务端口配置

**可扩展内容**:
- 公共类型定义
- 工具函数
- 常量配置
- 共享中间件

{
  "module": "nodenext",        // 使用最新的 Node.js 模块系统

## 微服务通信

### 服务间通信方式

1. **HTTP/REST API** (同步)
   - Gateway → User Service
   - Gateway → Post Service
   
2. **可扩展方案**:
   - 消息队列（RabbitMQ、Kafka）
   - gRPC（高性能 RPC）
   - 服务网格（Istio）

## 学习要点

### 1. 服务拆分原则

- **单一职责**: 每个服务专注一个业务领域
- **松耦合**: 服务间依赖最小化
- **高内聚**: 相关功能聚合在一起

### 2. API 网关的作用

- 统一入口点
- 请求路由
- 协议转换
- 安全认证
- 限流熔断

### 3. 服务发现

当前使用硬编码端口，生产环境可使用：
- Consul
- Eureka
- Kubernetes Service

### 4. 配置管理

- 使用 common 模块共享配置
- 环境变量管理敏感信息
- 配置中心（Apollo、Nacos）

## 开发流程

### 1. 安装依赖

```bash
pnpm install
```

### 2. 启动服务

```bash
# 启动用户服务
cd packages/user && node index.js

# 启动帖子服务
cd packages/post && node index.js

# 启动网关服务
cd packages/gateway && node index.js
```

### 3. 开发建议

- 先实现单个服务的核心功能
- 再通过 Gateway 整合
- 逐步添加服务间通信
- 考虑错误处理和重试机制
