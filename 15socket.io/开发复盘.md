## Socket.IO 聊天室项目复盘（开发逻辑）

这个项目是一个最小可用的“多人聊天室”Demo，核心目标是：

- **服务端**：维护房间成员列表、接收消息并按房间广播、同步房间在线人数
- **客户端**：连接 Socket.IO、加入房间、发送/接收消息、渲染 UI 与房间列表

---

### 目录与入口

- **服务端入口**：`server.js`
- **前端页面**：`index.html`
- **依赖**：`express` + `socket.io`
- **启动方式**：`npm start`（等价 `node server.js`）

---

### 整体运行链路（从启动到聊天）

#### 1）启动 HTTP + 静态页面

服务端使用 Express + Node `http`：

- `express()` 创建应用
- 设置 CORS 相关 header（便于跨域/本地调试）
- `express.static(__dirname)` 把当前目录静态托管出来
- `GET /` 返回 `index.html`
- `http.createServer(app)` 创建 HTTP Server

这样你可以直接打开：

- `http://localhost:<端口>/` 访问聊天页面

#### 2）把 Socket.IO 挂到同一个 HTTP Server

```js
const io = new Server(server, { cors: { origin: '*', methods: ['GET', 'POST'] } })
```

这一步的意义：

- Socket.IO 的握手、轮询、WebSocket 升级都走同一个端口
- 前端用 `io('http://localhost:<端口>')` 即可连接

#### 3）客户端连接成功后 join 房间

前端流程（`index.html`）：

- `prompt` 获取 `name` 和 `room`
- `const socket = io('http://localhost:<端口>')`
- `socket.on('connect', ...)` 触发后 `socket.emit('join', { name, room })`

服务端处理 join：

- `socket.join(room)`：把当前连接加入到 Socket.IO 的 room
- 把用户信息写入 `groupList[room]`
- 给当前用户发一条系统消息：`socket.emit('message', { user:'管理员', text:'xxx进入了房间' })`
- 同步房间列表：
  - 当前用户：`socket.emit('groupList', groupList)`
  - 其他用户：`socket.broadcast.emit('groupList', groupList)`

#### 4）客户端发送消息

前端：

- 输入框（contenteditable）里输入
- Enter 发送 / Shift+Enter 换行
- `socket.emit('message', { text, room, user: name })`

服务端：

- 收到 `message` 后，通过 `socket.broadcast.to(room).emit('message', { text, user })` 广播给**同房间的其他人**
- 发送者自己的 UI 更新由前端本地 `sendMessage(...)` 立即渲染（减少等待）

#### 5）断开连接（离线）

服务端监听 `disconnect`：

- 遍历 `groupList`，找出离线的 socket id
- 如果找到：
  - 向该用户所在 room 广播系统消息：`${name}离开了房间`
  - 从 `groupList[room]` 里剔除该用户
- 最后向所有人广播最新 `groupList`

---

### 服务端事件清单（你需要记住的“协议”）

服务端 `io.on('connection', socket => { ... })` 内部：

- **`join`**（客户端 -> 服务端）
  - 入参：`{ name, room }`
  - 行为：加入 room、写入 `groupList`、下发系统消息与 `groupList`
- **`message`**（客户端 -> 服务端）
  - 入参：`{ text, room, user }`
  - 行为：广播给同 room 的其他人（发送者本地渲染）
- **`disconnect`**（内置事件）
  - 行为：清理 `groupList`、广播系统消息、广播最新 `groupList`

服务端下发给客户端：

- **`message`**（服务端 -> 客户端）
  - 结构：`{ user, text }`
  - `user === '管理员'` 代表系统消息（进入/离开提示）
- **`groupList`**（服务端 -> 客户端）
  - 结构：`{ [roomName]: Array<{ name, room, id }> }`

### 前端渲染逻辑（UI 与数据如何绑定）

#### 1）为什么引入 jQuery

当前版本引入了 jQuery，主要用于：

- **更快获取元素**：`$('.ipt')`、`$('#sendBtn')`
- **事件绑定更简洁**：`$ipt.on('keydown', ...)`
- **拼 DOM 更方便**：`$('<div class="msg"></div>')` 然后 `.append(...)`

#### 2）消息渲染规则

- 普通消息：
  - 自己发送：`message.user === name` → 添加 `.mine` 样式（靠右）
  - 其他人：靠左
  - 消息内容使用 `.text(...)` 写入，避免 XSS/HTML 注入
- 系统消息：
  - `user === '管理员'` → 渲染为居中的 `.sys` 提示条

#### 3）房间列表渲染规则

- 收到 `groupList`：
  - 清空旧列表
  - 遍历每个 room：渲染 room 名 + 人数
  - 当前加入的 room 加 `.active` 高亮
  - 无数据则显示空状态

### 关键注意点（踩坑记录 & 复盘要点）

- **Express 5 通配符**：
  - `app.use('*', ...)` 在 Express 5（`path-to-regexp`）可能报错
  - 建议用 `app.use((req,res,next)=>...)` 做全局中间件

- **端口占用**：
  - 报错 `EADDRINUSE` 说明端口被占用，需要换端口或结束占用进程

- **端口要一致**：
  - 服务端 `server.listen(<端口>)`
  - 前端 `io('http://localhost:<端口>')`
  - 这两者不一致会导致“页面能打开但连接失败”

- **room 的作用**：
  - `socket.join(room)` 后，广播可用 `to(room)` 限定范围
  - 这就是“只给同房间的人发消息”的关键

- **groupList 的数据结构**：
  - 当前实现是内存对象，重启服务端会丢失
  - 适合 Demo；要做正式产品需要持久化（Redis/DB）或在多实例间同步

---

### 你下次想扩展时的路线图

- **点击房间切换**：点击左侧房间 → `join` 新 room、离开旧 room、清屏并重新拉群成员
- **消息历史**：服务端保存最近 N 条消息；用户 join 时下发 `history`
- **昵称校验**：空昵称/重名处理
- **“我正在输入”状态**：`typing` 事件广播
- **多实例部署**：用 `@socket.io/redis-adapter` 解决跨进程 room 广播
