# 短链接服务实现思路

## 核心原理

短链接服务通过生成唯一的短码(如 `abc123`)来映射长 URL,当用户访问短链接时,服务器查询数据库找到对应的长 URL 并重定向。

## 技术栈

- Express - Web 框架
- Knex.js - SQL 查询构建器
- MySQL2 - 数据库驱动
- shortid - 短码生成库

## 数据库设计

```sql
CREATE TABLE short (
    id VARCHAR(255) PRIMARY KEY,      -- 短码(唯一标识)
    url TEXT NOT NULL,                -- 原始长URL
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP  -- 创建时间
);
```

## 实现流程

### 1. 创建短链接

**接口:** `POST /create_url`

**流程:**
1. 接收原始 URL
2. 使用 `shortid.generate()` 生成唯一短码
3. 将短码和 URL 存入数据库
4. 返回完整的短链接地址

```javascript
app.post('/create_url', async (req, res) => {
    const { url } = req.body
    const id = shortid.generate()  // 生成短码,如 '2mEqjYpKy'
    await db('short').insert({ id, url })
    res.send(`http://localhost:3000/${id}`)
})
```

### 2. 短链接重定向

**接口:** `GET /:id`

**流程:**
1. 从 URL 路径中提取短码
2. 在数据库中查询对应的原始 URL
3. 如果找到,使用 `res.redirect()` 重定向到原始 URL
4. 如果未找到,返回 404 错误

```javascript
app.get('/:id', async (req, res) => {
    const { id } = req.params
    const result = await db('short').where({ id }).first()
    if (result) {
        res.redirect(result.url)  // 302 重定向
    } else {
        res.status(404).send('URL not found')
    }
})
```

## 关键点

### 1. 短码生成策略

使用 `shortid` 库生成短码的优势:
- 生成的 ID 短小(通常 7-14 个字符)
- URL 友好(不包含特殊字符)
- 碰撞概率极低
- 无需自增 ID,适合分布式系统

### 2. 数据库查询优化

- `id` 字段设为主键,查询效率高
- 使用 `.first()` 直接返回单条记录,而不是数组
- 可以添加索引优化查询性能

### 3. HTTP 重定向

使用 `res.redirect()` 实现 302 临时重定向:
- 浏览器会自动跳转到目标 URL
- 保留短链接的访问统计能力
- 可以后续升级为 301 永久重定向

## 可扩展功能

1. **访问统计** - 记录每次访问的时间、IP、User-Agent
2. **过期时间** - 设置短链接有效期
3. **自定义短码** - 允许用户指定短码
4. **访问限制** - 设置访问次数限制或密码保护
5. **二维码生成** - 为短链接生成二维码
6. **防滥用** - IP 限流、验证码验证

## 常见问题

### 1. 短码冲突怎么办?

`shortid` 的碰撞概率极低,但可以添加重试逻辑:
```javascript
let id, exists;
do {
    id = shortid.generate();
    exists = await db('short').where({ id }).first();
} while (exists);
```

### 2. 如何提高性能?

- 添加 Redis 缓存热门短链接
- 使用连接池优化数据库连接
- 对 `id` 字段建立索引(主键自动索引)

### 3. 数据库初始化

执行 `init-db.sql` 创建数据库和表:
```bash
mysql -u root -p < init-db.sql
```
