## 背景

项目目录：`e:\github\study\nodejs\12lua\`

现象：
- 启动后控制台刷屏报错：`connect ECONNREFUSED ::1:6379`
- 访问 `http://localhost:3000/lottery` 返回“请稍后重试！”
- 编辑器里 `test.lua` 出现类似 `@test.lua (14)` 的提示/告警

## 根因总结

### 1) Redis 连接失败（IPv6 `::1`）

`ioredis` 在 `new Redis()` 默认会连接 `localhost:6379`，在部分 Windows 环境下 `localhost` 会优先解析到 IPv6 回环 `::1`。

如果本机 Redis 没启动、或只监听 IPv4（`127.0.0.1`），就会出现：
- `Error: connect ECONNREFUSED ::1:6379`
- 并且因为没有处理 error event，会刷 “Unhandled error event”

### 2) 端口 3000 被旧进程占用（导致访问到了“旧服务”）

当 `:3000` 已被旧的 Node 进程监听时，你即使改了代码/重启了新的进程，浏览器访问的还是旧进程返回的结果，表现为：
- 你以为“已生效”的改动没有生效
- `/lottery` 一直返回“请稍后重试！”

可用命令定位占用：

```powershell
netstat -ano | findstr ":3000"
```

### 3) `test.lua` 的编辑器告警是“静态检查误报”

Redis 的 Lua 环境里，`KEYS` / `ARGV` / `redis` 是内置全局变量；但 Lua 语言服务做静态检查时会认为它们未定义，从而给出 warning（并非运行时错误）。

## 做过的代码改动

## `index.js` / `test.lua` 实现说明（做了什么，怎么做的）

### 整体目标

实现一个简单的“抽奖接口” `GET /lottery`，并通过 **Redis + Lua 脚本** 做限流：
- 在一个时间窗口内（默认 30 秒），同一个 key 最多允许成功 \(N\) 次（默认 5 次）
- 超过阈值后返回“请稍后重试！”

### `index.js` 做了什么

- **启动一个 HTTP 服务**
  - 使用 `express` 启动端口 `3000`
- **连接 Redis**
  - 默认连接 `127.0.0.1:6379`（IPv4）
  - 支持用环境变量覆盖：
    - `REDIS_URL`（优先级最高）
    - `REDIS_HOST` / `REDIS_PORT`
  - 监听 `redis.on('error')`，把连接/执行错误打印出来，避免未处理错误事件刷屏
- **读取 Lua 脚本**
  - 启动时读取 `./test.lua` 的内容字符串，后续在接口里执行
- **提供接口：`GET /lottery`**
  - 每次请求都会调用 `redis.eval(lua, 1, KEY, CHANGE, TIME, cb)`
  - 参数含义（对应 Redis `EVAL`）：
    - `lua`：Lua 脚本内容
    - `1`：`KEYS` 的数量（这里只有 1 个 key）
    - `KEY`：脚本内的 `KEYS[1]`，这里固定为 `"lottery"`
    - `CHANGE`：脚本内的 `ARGV[1]`，这里是限流阈值（默认 5）
    - `TIME`：脚本内的 `ARGV[2]`，这里是时间窗口秒数（默认 30）
  - 回包逻辑：
    - 脚本返回 `1`：返回“抽奖成功”
    - 脚本返回 `0`：返回“请稍后重试！”
    - Redis 执行出错：返回 `500` + “Redis异常，请检查服务是否可用”

### `test.lua` 做了什么

`test.lua` 是 Redis 侧执行的 Lua 脚本，核心是“计数 + 过期时间”实现滑动不精确的固定窗口限流。

- **入参约定（由 `index.js` 的 `eval` 传入）**
  - `KEYS[1]`：计数用的 Redis key（这里是 `"lottery"`）
  - `ARGV[1]`：`limit` 阈值（这里是 5）
  - `ARGV[2]`：`interval` 时间窗口（这里是 30 秒）
- **执行逻辑**
  - `count = GET key`（不存在按 0 处理）
  - 如果 `count > limit`：
    - 返回 `0`（表示被限流/失败）
  - 否则：
    - `INCR key`
    - `EXPIRE key interval`（设置过期时间，窗口结束自动清零）
    - 返回 `1`（表示放行/成功）

> 注意：脚本里用的是 `count > limit`，这意味着当 `limit=5` 时，实际允许返回成功的次数会是 6 次（0,1,2,3,4,5 都会走到 else 并 incr）。
> 如果你希望严格“最多 5 次”，条件应改为 `count >= limit`（这里只是说明，不代表必须修改）。

### 为什么 `test.lua` 在编辑器会有提示

Lua 语言服务会把 `KEYS` / `ARGV` / `redis` 当作未定义全局变量（普通 Lua 语境下确实未定义），但它们是 Redis Lua 环境内置的。
所以这是静态检查误报，已通过 `---@diagnostic disable: undefined-global` 屏蔽。

### `index.js`

- **Redis 连接改为优先 IPv4**
  - 默认 `host: 127.0.0.1, port: 6379`
  - 支持环境变量：
    - `REDIS_URL`（优先级最高）
    - `REDIS_HOST`
    - `REDIS_PORT`
- **增加 `redis.on('error', ...)`**：避免 “Unhandled error event”，输出更可读的错误信息
- **/lottery 处理增强**
  - `redis.eval` 报错时返回 `500`（不再把所有异常都当成“请稍后重试”）
  - 用 `Number(result) === 1` 判断结果（兼容返回值类型差异）
  - 增加日志：`[lottery] eval result: ...`

### `package.json`

- 新增 `scripts.start`：

```json
"start": "node ."
```

### `test.lua`

- 添加一行关闭静态诊断误报：

```lua
---@diagnostic disable: undefined-global
```

## 常用验证命令

### 1) 检查 Redis 是否在 IPv4 上监听

```powershell
Test-NetConnection 127.0.0.1 -Port 6379 | Format-List -Property ComputerName,RemoteAddress,RemotePort,TcpTestSucceeded
```

### 2) 启动项目

```powershell
npm start
```

### 3) 测试接口

```powershell
Invoke-WebRequest -UseBasicParsing "http://localhost:3000/lottery" | Select-Object -ExpandProperty Content
```

### 4) 排查端口占用（关键）

```powershell
netstat -ano | findstr ":3000"
# 停掉占用端口的 PID
Stop-Process -Id <PID> -Force
```
