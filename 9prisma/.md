# Prisma 项目介绍

## 初始化 TypeScript 项目

```bash
npm init -y
npm install typescript tsx @types/node --save-dev
npx tsc --init
```

## 安装 Prisma

```bash
npm install prisma @types/node --save-dev
npm install @prisma/client @prisma/adapter-mariadb dotenv
```

以下是每个软件包的功能：

- `prisma` - Prisma CLI 用于运行诸如 `prisma init`、`prisma migrate` 和 `prisma generate` 之类的命令
- `@prisma/client` - 用于查询数据库的 Prisma 客户端库
- `@prisma/adapter-mariadb` - 将 Prisma Client 连接到数据库的 MySQL/MariaDB 驱动程序适配器
- `dotenv` - 从 `.env` 文件中加载环境变量

## 初始化 Prisma ORM

现在您可以通过在命令前加上前缀 `npx` 来调用 Prisma CLI：

```bash
npx prisma
```

接下来，使用以下命令创建 Prisma Schema 文件，从而设置 Prisma ORM 项目：

```bash
npx prisma init --datasource-provider mysql --output ../generated/prisma
```

这条命令会执行以下几项操作：

- 创建一个 `prisma/` 目录，其中包含一个 `schema.prisma` 文件，该文件包含您的数据库连接和模式模型
- 在根目录下创建一个 `.env` 文件用于存储环境变量
- 创建 `prisma.config.ts` Prisma 配置文件

## 定义数据模型

```prisma
generator client {
  provider = "prisma-client"
  output   = "../generated/prisma"
}

datasource db {
  provider = "mysql"
}

model User { 
  id    Int     @id @default(autoincrement()) 
  email String  @unique
  name  String?
  posts Post[]
} 

model Post { 
  id        Int     @id @default(autoincrement()) 
  title     String
  content   String? @db.Text
  published Boolean @default(false) 
  author    User    @relation(fields: [authorId], references: [id]) 
  authorId  Int
}
```

## 创建并应用您的第一个迁移

### 创建您的第一个迁移以设置数据库表

```bash
npx prisma migrate dev --name init
```

这个命令主要用于开发环境。它的作用是将你在 `schema.prisma` 文件中定义的数据模型同步到数据库中。

命令拆解：

- `migrate` - 表示执行数据库迁移操作
- `dev` - 表示在开发模式下运行。这是关键，它在开发环境中提供了一些便利功能（如自动检测漂移并重置数据库）
- `--name init` - 给这次迁移起个名字。Prisma 会在 `prisma/migrations` 目录下创建一个文件夹，名字格式通常为 `时间戳_init`。这个名字方便你识别这次迁移的内容（例如 `init` 通常用于项目第一次初始化）

具体执行步骤：

1. 读取 Schema：读取 `schema.prisma` 文件
2. 创建迁移文件：根据 Schema 的变化，生成对应的 SQL 文件（保存在 `prisma/migrations/` 目录）
3. 应用迁移：将生成的 SQL 执行到你的开发数据库中（创建表、添加列等）
4. 检测漂移 (Drift Detection)：检查数据库当前状态是否与 `schema.prisma` 或之前的迁移记录不一致。如果不一致（例如你手动在数据库里改了表结构），`dev` 模式会提示你是否重置数据库以保持干净的状态
5. 生成 Client：执行完迁移后，它通常会自动触发 `prisma generate`，确保代码也是最新的

适用场景：

- 开发过程中，修改了 `schema.prisma`（加了模型、改了字段类型）
- 项目刚开始，需要初始化数据库表结构

### 生成 Prisma 客户端

此命令根据您的架构创建数据库表。现在运行以下命令生成 Prisma 客户端：

```bash
npx prisma generate
```

这个命令用于生成 Prisma Client。它不直接操作数据库。

具体作用：

1. 读取 Schema：读取 `schema.prisma` 文件
2. 生成代码：根据 Schema 生成类型安全的数据库查询客户端代码（通常位于 `node_modules/.prisma/client` 或你配置的 `output` 目录）
3. 提供类型提示：生成的代码包含了 TypeScript 类型定义。这使得你在写代码（如 `prisma.user.findMany()`）时，IDE 能提供智能提示（IntelliSense）和编译时类型检查

为什么需要它？

Prisma Client 不是静态存在的，它是根据你的数据库模型动态生成的。如果你修改了 `schema.prisma`（例如加了一个 `age` 字段），但不运行 `generate`，你的代码里就不知道 `age` 的存在，TypeScript 会报错，或者运行时出错。

适用场景：

- 修改了 `schema.prisma` 后
- 生产环境构建时：在 Docker 构建或 CI/CD 流程中，通常先运行 `generate` 生成客户端，然后再部署代码（注意：生产环境更新数据库通常用 `prisma migrate deploy`，而不是 `dev`）
- 当你只修改了 Schema 中的某些不影响数据库结构的配置（如添加 `@@map` 映射），不需要迁移数据库，但需要更新代码类型时
